## 已完成

- 事件处理与可靠性增强：
  - 已新增事件处理：`im.message.recalled_v1`（撤回回滚）、`im.message.reaction.deleted_v1`（取消点赞回滚）。

- 去重策略升级：

- 标签识别健壮性增强：
  - 标签识别已支持全角井号归一化：`＃ -> #`。
  - 已支持标签尾部标点清洗（如 `。`、`，`、`！` 等）。

## 当前差异（相对最新需求）

- 唯一冲突点：
  - 当前实现对“未知标签”采用“默认文档 fallback 归档”。
  - 最新需求要求“仅已知标签归档文档；未知标签/无标签不归档文档，但都要计入活跃值”。

## 下一步计划

- 调整标签归档路由规则：
  - 仅当标签命中 `TAG_MAPPING` 且对应 token 已配置时，写入标签文档。
  - 未知标签与无标签统一不写文档（保留日志说明原因）。
- 保持活跃值统计全量计入：
  - 目标群内消息不受标签命中结果影响，均计入活跃值累计。
  - 归档链路失败/跳过不阻断统计链路。
- 日志语义对齐：
  - 明确区分“已知标签归档成功”“未知标签不归档”“无标签不归档”，并强调“已计入活跃值”。
