## 本次目标

- 修正每周 `Pin` 汇总卡的长帖展示问题，避免帖子被扁平化成难读的大段纯文本。
- 让折叠前预览覆盖全部加精帖子，而不是只显示前两条。
- 消除机器人外层序号与帖子正文内部编号叠加造成的视觉冲突。

## 已完成

- `Pin` 汇总卡改为基于 `raw_content` 单独渲染卡片 markdown：
  - 不再只依赖扁平化的 `content` 字段。
  - `processed_items` 新增保留 `raw_content`，仅卡片展示层使用。
- 富文本卡片展示能力已补齐到“文本富文本”为主：
  - 保留段落和空行。
  - 支持列表、粗体、斜体、删除线、链接、`@`。
  - 图片和附件不直接嵌入卡片，统一显示为 `[图片]` / `[附件]` 占位。
- 预览区改为展示全部帖子：
  - 有几条 `Pin` 就显示几条预览。
  - 每条预览最多 `50` 字。
  - 预览截断基于“可见纯文本”，不直接截 raw markdown，避免把链接或样式语法截坏。
- 帖子外层展示改成“分隔标题”样式：
  - 统一格式为 `**【帖子一】发送者（时间）**`
  - 删除外层 `1. 2. 3.` 阿拉伯序号。
  - 多条帖子之间插入 `hr` 分隔。
- 折叠阈值仍为 `500`，但判定口径已改成“完整正文的可见文本长度”，避免受 markdown 转义字符影响。

## 关键方法与技巧沉淀

- 飞书卡片富文本不要复用共享的“纯文本提取器”：
  - `MetricsCalculator.extract_text_from_content()` 适合统计和归档表写入。
  - 卡片展示需要单独的 `raw_content -> card markdown` 渲染器，否则段落、链接和样式都会被抹平。
- 预览摘要不要直接截 markdown 源字符串：
  - 正确顺序应为：`full_markdown -> plain_text -> truncate`
  - 这样可以避免出现半截链接、半截加粗符号或多余反斜杠。
- 外层帖子编号和正文内部列表容易冲突：
  - 对于本身就常含 `1. 2. 3.` 的帖子，推荐用“分隔标题”替代外层数字列表。
  - 标题承担分隔职责，正文内部继续保留原始编号。
- 富文本转卡片 markdown 时要先做文本转义，再叠加样式：
  - 普通文本应先转义 `# [ ] ( ) * _ ~` 等控制字符。
  - 然后再按 `style/un_style` 包装成 `**粗体**`、`*斜体*`、`~~删除线~~`。
  - 否则像 `#个人思考` 这类原文内容会被误渲染成 markdown 标题。
- 对图片/附件这类难直接复用的消息资源，先用文本占位是更稳的默认策略：
  - 避免为了追求 1:1 还原，把消息卡片链路变成额外的文件上传链路。
