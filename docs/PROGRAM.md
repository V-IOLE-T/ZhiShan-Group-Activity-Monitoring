## 2. 功能总览（按业务链路）

### 2.1 实时监听与活跃度统计

- 通过飞书长连接接收：
  - 消息事件
  - 表情回复（点赞）事件
- 指标包含：
  - 发言次数、发言字数、被回复数、被@次数、发起话题数、点赞数、被点赞数、被 Pin 次数
- 统计写入 Bitable，按月累积并计算活跃度分数
- 已实现 LRU 去重和批量更新阈值（当前 `BATCH_UPDATE_THRESHOLD=3`）

### 2.2 群消息归档到文档（Docx）

- 支持按 `#标签` 路由到不同目标文档
- 支持文本、富文本、图片/文件附件处理与插入
- 回复消息支持插入到分割线前，保持上下文阅读性

### 2.3 Pin 能力（当前生产）

- 每周 Pin 审计（调度器）：
  - 每周一 `09:00` 触发
  - 检查“上周新增 Pin（按 Pin 操作时间）”
  - 若有新增：发送 1 张卡片（标题“上周加精”）
  - 卡片正文仅包含编号明细，展示每条完整内容（不做摘要截断）
  - 自动执行：归档 Pin 记录、更新被 Pin 次数、写入精华文档（已配置时）
  - 若无新增：不发送任何消息
  - 去重机制：同一消息仅处理一次（持久化到 `.processed_daily_pins.txt`）

### 2.4 月度归档

- 调度器每日 `02:00` 检查
- 仅在每月 1 号执行：将当月统计表复制到历史表并清空当月表

### 2.5 单聊智能回复（reply_card）

- 识别文档链接，调用 MCP 获取内容摘要
- 生成图片卡片并回传
- 纯文本也可生成简洁图片回复

### 2.6 可观测性

- 健康检查服务：
  - `/health`
  - `/status`
  - `/metrics`（Prometheus 风格）
- 日志轮转已配置，避免日志无限增长

## 4. 运行稳定性风险（建议后续修复）

### 风险 1：环境校验工具的加载路径依赖调用方式

- `env_validator.py` 默认 `load_dotenv()`，直接独立调用时可能读不到 `config/.env`。
- 目前主入口可通过 `long_connection_listener.py` 的加载逻辑规避，但独立脚本/命令调用容易产生“伪故障”。

### 风险 2：缺少在线集成回归

- 当前测试以单元测试为主，对外部长连接事件流、调度线程与真实数据写入的联动验证不足。

## 5. 推荐运行与验收流程

1. 启动前
- 确认 `config/.env` 完整
- 运行：`python -m unittest discover tests -v`

2. 启动服务
- 命令：`python main.py`
- 观察启动日志是否出现健康检查地址和监听启动标记

3. 运行中巡检
- `GET /health` 返回 healthy
- 日志中无持续异常重连
- 关键业务事件（消息、点赞、每周 Pin 审计）按预期落表/发卡

4. 周期性检查
- 每周一检查 09:00 Pin 审计执行日志
- 每月 1 号检查月度归档结果

## 6. 与 API 调用控制相关的现状说明

- 已移除秒级 Pin 轮询，Pin 相关 API 调用改为每周定时触发
- 本文档未修改 API 调用逻辑，仅记录现状与稳定性
